PNG/JPEG image uploads now have a structured overlay rendering path. Image preprocessing uses Tesseract TSV output to write line-level OCR block coordinates beside the extracted text, the agent prompt requires `translations.json` for structured image artifacts, and the worker renders `<source-stem>_translated.pdf` by placing translated text boxes over the original image. The `[원문]` section reuses the untouched source image page rather than reconstructing OCR text. If structured image translations are unavailable, the worker falls back to the existing text-based PDF renderer.

Image overlay rendering is now collision-aware. The renderer lays out OCR translation boxes before drawing, moves overlapping boxes downward when possible, draws all translucent backgrounds before drawing any translated text, and then draws text as the final layer so a later opaque OCR box cannot hide earlier translated text.

Image overlay labels are now single-line-height boxes without strokes. The worker sizes each translated label background to the rendered text line-height, removes the previous gray outline, keeps backgrounds translucent, and fits one-line text horizontally where possible. ## Durable Decisions

- Use Telegram as the single capture, command, and notify channel.
- Use Telegram Local Bot API Server for large files.
- Do not include Dropbox in V1.
- Use npm workspaces initially for harness compatibility.
- Use deterministic TypeScript code for queue, state, file import, retry, permissions, and notifications.
- Use `packages/db` as the SQL boundary; app code should call repository functions rather than writing SQL inline.
- Use `packages/capture` as the Telegram update-to-job orchestration boundary.
- Use `packages/importer` as the Telegram file import boundary; never process files in-place from Bot API storage.
- Use `packages/vault` as the immutable raw bundle writing boundary.
- Use `packages/rtzr` as the RTZR STT boundary for auth, submit/poll, transcript artifacts, and ffmpeg conversion helpers.
- Use `packages/sensevoice` as the on-demand local SenseVoice STT boundary; invoke it per job rather than as a resident service.
- Use `packages/wiki-adapter` as the LLM/wiki command boundary, guarded by a write lock and raw snapshot checks.
- Use `packages/output-store` as the runtime-only output delivery boundary; output files are TTL cache artifacts, not source-of-truth vault content.
- Use `packages/preprocessors` and `packages/language-detector` for deterministic pre-agent text collection and translation-needed checks.
- Use `packages/agent-adapter` as the local agent command boundary; adapter workspaces may write outputs only and must not mutate `raw/**`.
- Use `packages/operator` for Telegram status/retry/cancel and notification/report text.
- Keep agents as Markdown/JSON translation workers for document post-processing; worker-owned renderers create and validate DOCX/PDF deliverables, and uploaded DOCX sources use block-id `translations.json` replacement to preserve the original package/template when possible.
- Use RTZR STT for audio and voice files.
- Write immutable Obsidian raw bundles under `raw/<date>/<source_id>/`.
- Let the LLM wiki adapter update `wiki/**` only, never `raw/**`.
- LLMwiki may read only `source.md`, `manifest.yaml`, and manifest-declared canonical wiki inputs by default.
- Treat `_translated.*`, image overlay PDFs, transcript DOCX files, and `runtime/outputs/**` as user-facing deliverables, not wiki source authority.
